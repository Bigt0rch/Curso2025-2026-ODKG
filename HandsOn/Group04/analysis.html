Group 04 — Data analysis, licensing and resource naming strategy

1. Datasets

  • flux_daily_mov.csv — daily aggregated movements/flows between zones (columns: date, origin_zone_id, dest_zone_id, trips, maybe vehicle_type).
  • zones.csv — zone definitions (columns: zone_id, zone_name, centroid_lat, centroid_lon, type).

2. Analysis of the data

We inspected the two CSV files to characterise their schema, cardinalities and potential data quality issues. Below is a compact summary based on sampling and expected fields.

Quality & ranges

  • Missing zone coordinates: a small fraction of zones may lack centroid coordinates. We will still expose the zone URI and omit geometry when absent.
  • Trips zero values: flows with zero trips can be omitted from the RDF instantiation or included as explicit zero values — we choose to include only positive flows in the sample data to keep examples compact.
  • Dates: ensure ISO 8601 YYYY-MM-DD formatting.

3. Licensing and provenance

Publisher and rights holder: the datasets come from the course hands-on materials (instructors). Suggested licence: CC BY 4.0 (Creative Commons Attribution 4.0 International) — allows reuse with attribution. If the original source imposes different restrictions, choose the most restrictive licence required by the data owner.

Provenance metadata to include when publishing the RDF dataset:

  • dc:publisher — course/instructor name
  • dc:license — http://creativecommons.org/licenses/by/4.0/
  • prov:wasDerivedFrom — original CSV file names and commit hash (if published from this repository)

4. Resource naming strategy

Base URI for the group's resources:

https://data.example.org/odkg/group04/

URI patterns:

  • Zone: https://data.example.org/odkg/group04/zone/{zone_id}
  • Movement (flow): https://data.example.org/odkg/group04/flow/{date}/{origin_zone_id}/{dest_zone_id}
  • Dataset (metadata): https://data.example.org/odkg/group04/dataset/{dataset-id}

Examples:

https://data.example.org/odkg/group04/zone/12
https://data.example.org/odkg/group04/flow/2024-05-01/12/34

5. Ontology (summary)

We define a lightweight ontology with two main classes and a small set of properties.

  • Class Zone — represents spatial units (zones.csv).
  • Class Flow — represents aggregated movement between two zones on a given date.
Key properties (short list):
  • ex:hasOrigin (Flow → Zone)
  • ex:hasDestination (Flow → Zone)
  • ex:trips (Flow → xsd:integer)
  • ex:date (Flow → xsd:date)
  • ex:zoneName, geo:lat, geo:long for Zone properties

6. Mapping notes

When converting CSV to RDF:

  • Use the URI patterns above to create subjects for zones and flows.
  • Map numeric values to xsd:integer and dates to xsd:date.
  • Keep provenance triples linking flows to the original CSV row via prov:wasDerivedFrom.

7. Short sample SPARQL examples

Count total trips between two zones in a time range (example):

PREFIX ex: <https://data.example.org/odkg/group04/ontology/#>
SELECT (SUM(?t) AS ?totalTrips) WHERE {
  ?flow a ex:Flow ; ex:hasOrigin ?o ; ex:hasDestination ?d ; ex:trips ?t ; ex:date ?date .
  ?o ex:zoneId "12" .
  ?d ex:zoneId "34" .
  FILTER (?date >= "2024-05-01"^^xsd:date && ?date <= "2024-05-31"^^xsd:date)
}
