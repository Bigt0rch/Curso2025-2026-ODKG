Dataset Requirements - Madrid's Daily Metereology

Requirement 1: Data are in one of the smart city domains

The dataset belongs mainly to the Smart Environment domain of Smart Cities, with intersections in Smart Living/Health, since it tracks temperature using geolocated stations over time.

Requirement 2: Data are available as a CSV file

The selected dataset is available as a CSV file, which is located inside the folder "CSV FILES" uploaded for the first assignment

Requirement 3: Data have an open license so they can be published

The open license of this data can be found in the following link: https://datos.madrid.es/egob/catalogo/aviso-legal

Requirement 4: Data can be easily linked with generic real-world entities

The dataset can be easily linked with real-world entities such as geographic areas (province, municipality, station) and time (days, months, years). Fields like “PROVINCIA”, “MUNICIPIO” and “ESTACION” directly map to real-world locations, referring to administrative areas and physical meteorological stations. Similarly, “ANO”, “MES”, “D01”, “D02”, etc., represent actual calendar dates, making the dataset directly relatable to external sources such as other meteorological records, climate models, traffic patterns or public health data for the same period. In this dataset, which contains meteorological information for Madrid, several attributes can be directly associated with generic real-world entities:

  • Measurement stations, such as Plaza España, Escuelas Aguirre, or Cuatro Caminos, which correspond to physical monitoring sites across the city.
  • Geographical entities, including province, municipality, and neighborhood, which situate each measurement station in its real location.
  • Meteorological variables, such as temperature, humidity, wind direction, or precipitation, which represent physical environmental conditions that can be observed and compared with global standards.

Requirement 5: Documentation exists for the data (optional)

The dataset is fully documented by the Dirección General de Sostenibilidad y Control Ambiental. It explains the meaning of each field (province, municipality, station, parameter, measurement technique, year, month, day), the coding of meteorological variables (e.g., temperature, humidity, wind speed, precipitation), and the mapping of station codes to real locations. It also specifies the validation rules, indicating that only data marked with the code “V” are valid. This ensures the dataset is standardized, interpretable, and suitable for integration into Smart City applications. This documentation is available at: https://datos.madrid.es/FWProjects/egob/Catalogo/MedioAmbiente/DatosMeteorologicos/Ficheros/Interpretaci%C3%B3n_datos_meteorologicos.pdf
